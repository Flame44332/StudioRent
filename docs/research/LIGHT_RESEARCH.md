# Light Research Sprint

Дата: 2026-06-07

## Быстрые выводы

- Telegram Mini App технически является web-приложением внутри Telegram, поэтому для MVP логичен web stack, а не мобильный SDK.
- Яндекс.Карты подходят через JavaScript API v3; Geocoder понадобится для адресов и координат студий.
- Каталог студий лучше держать в своей базе, а не строить MVP вокруг поиска организаций Яндекса: так проще контролировать карточки, цены, слоты и бронирование.
- Для студий главный B2B-угол: меньше ручных сообщений, понятный календарь, витрина с фото/оборудованием, заявки из Telegram.
- Для клиента главный B2C-угол: быстро сравнить студии по району, цене, оборудованию и свободному времени.

## Первые гипотезы MVP

- Темный интерфейс в стиле BeatChain хорошо совпадает с Telegram Mini App контекстом.
- Главная должна быть не лендингом, а рабочим каталогом: поиск, карта, подборки, быстрые фильтры.
- На первом этапе реальную оплату можно заменить mock-экраном, но модель статусов брони сразу делать как под настоящую оплату.
- Студии на старте лучше добавлять вручную, чтобы быстрее получить качественные карточки и не строить модерацию.

## Что проверить глубже позже

- Сколько активных студий звукозаписи в Москве и какие у них типовые цены.
- Какие фильтры реально важны артистам: метро, цена, оборудование, инженер, жанр, акустика, парковка.
- Готовность студий отдавать слоты во внешнюю систему.
- Модель комиссии и возможный конфликт с прямыми бронированиями.
