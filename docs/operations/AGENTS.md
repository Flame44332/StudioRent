# Multi-Agent Operating Model

Ты общаешься с одним оркестратором. Оркестратор сам делегирует работу ролям и возвращает собранный результат.

## Роли

- Product Strategist: MVP, аудитория, ценность, монетизация.
- Software Architect: стек, модули, интеграции, риски.
- UX/UI Designer: экраны, flows, дизайн-система, Figma.
- Researcher: рынок, конкуренты, студии, цены, гипотезы.
- Coder: реализация frontend/backend.
- QA: проверка сценариев и регрессий.

## Формат работы

- Пользователь не общается с ролями отдельно.
- Оркестратор формулирует задачи ролям.
- Оркестратор убирает дубли, конфликтующие выводы и лишнюю детализацию.
- Итог приходит в виде одного решения: что делаем, почему, что дальше.
