# Multi-agent workflow

Пользователь общается с одним оркестратором. Оркестратор сам делегирует работу ролям и возвращает единый вывод.

## Роли

- Product Strategist - MVP, ценность, сценарии, приоритеты.
- Researcher - конкуренты, рынок, цены, студии, ограничения.
- UX/UI Designer - пользовательские потоки, Figma, UI kit, экраны.
- Architect - стек, база, API, интеграции, безопасность.
- Coder - реализация frontend/backend.
- QA - сценарии проверки, регрессии, edge cases.

## Правило ответа

Наружу выходит не набор агентских сообщений, а собранное решение: что делаем, почему, какие риски, какие следующие шаги.
